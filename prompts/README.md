# README for Prompts Directory

This directory contains all the prompts and communication context from the initial project setup phase.

## Directory Structure

- `system/` - System-level prompts defining project context, architecture, and guidelines
- `user/` - User requirements and requests that guided the framework creation  
- `examples/` - Implementation patterns and examples derived from our communication

## Purpose

These prompts serve as:
1. **Historical reference** - Documenting the original requirements and decisions
2. **Future guidance** - Helping maintain consistency during implementation
3. **Onboarding material** - Helping new team members understand project context
4. **AI agent instructions** - Providing context for future AI-assisted development

## Usage

When implementing new features or components, refer to these prompts to ensure alignment with:
- Original architectural decisions
- Technology stack choices  
- Development priorities
- Performance requirements
- Implementation patterns
